🔎 ১. Secret Leak হয়েছে কিনা বুঝবেন কিভাবে

Secret leak detect করতে দুটো জায়গা চেক করতে হবে — Local Git History এবং Remote GitHub।

📁 Local Git History চেক করুন

নিচের commands দিয়ে দেখুন কোনো sensitive keyword আপনার পুরো commit history-তে আছে কিনা:

PowerShell — যেকোনো keyword দিয়ে খুঁজুন
# "password" আছে কিনা খোঁজা git log -p --all | Select-String -Pattern "password" -Context 2 # "DATABASE_URL" আছে কিনা খোঁজা git log -p --all | Select-String -Pattern "DATABASE_URL" -Context 2 # "SECRET" বা "API_KEY" খোঁজা git log -p --all | Select-String -Pattern "SECRET|API_KEY|TOKEN" -Context 2
PowerShell — নির্দিষ্ট ফাইলটা কোন commit-এ আছে
# কোনো ফাইল কখনো commit হয়েছিল কিনা (এখন deleted হলেও দেখাবে) git log --all --full-history -- "filename.md" git log --all --full-history -- ".env" git log --all --full-history -- "*.env"
PowerShell — কোন commit-এ কী ছিল দেখুন
# একটা নির্দিষ্ট commit-এর সব changes দেখুন git show COMMIT_HASH # একটা commit-এ নির্দিষ্ট ফাইলের content দেখুন git show COMMIT_HASH:"path/to/file.md"
PowerShell — সব branch-এর সব file list দেখুন
# পুরো history-তে কোন কোন ফাইল ছিল (deleted সহ) git log --all --name-only --pretty=format: | sort -u | Select-String ".env" # বড় ফাইলগুলো দেখুন (sensitive docs বড় হয়) git rev-list --objects --all | git cat-file --batch-check="%(objecttype) %(objectname) %(objectsize) %(rest)" | Where-Object { $_ -match "^blob" } | Sort-Object { [int]($_.Split()[2]) } -Descending | Select-Object -First 20

🌐 Remote GitHub-এ চেক করুন

⚠️ GitHub-এ search করলে শুধু current code দেখায়। Commit history-তে থাকা পুরানো secrets ধরা পড়ে না — তাই local check বেশি গুরুত্বপূর্ণ।
Browser — GitHub Code Search
# GitHub-এ এই URL-এ যান (repo-তে keyword search) https://github.com/YOUR_USERNAME/YOUR_REPO/search?q=DATABASE_URL https://github.com/YOUR_USERNAME/YOUR_REPO/search?q=password https://github.com/YOUR_USERNAME/YOUR_REPO/search?q=SECRET
Browser — GitHub Commits ব্রাউজ করুন
# সব commits দেখুন https://github.com/YOUR_USERNAME/YOUR_REPO/commits/main # নির্দিষ্ট ফাইলের history দেখুন https://github.com/YOUR_USERNAME/YOUR_REPO/commits/main/path/to/file
PowerShell — Remote-এ কোন branches আছে দেখুন
# Remote-এর সব branches git ls-remote --heads origin # Remote থেকে সব fetch করুন তারপর local-এ check করুন git fetch --all git log --all --full-history -- ".env"
✅ কিভাবে বুঝবেন Leak হয়েছে:
• git log --all --full-history -- "filename" এ output আসলে — ওই ফাইল কোনো commit-এ ছিল
• GitHub থেকে Neon/GitHub-এর মতো security email পেলে
• git log -p --all | Select-String "password" এ কিছু দেখালে — history-তে password আছে

⚡ ২. Secret পেলে সঠিক Workflow — কী আগে, কী পরে

🚫 সবচেয়ে বড় ভুল: File delete করে বা .gitignore-এ add করে সরাসরি push করা। এতে file টা Git History-তে থেকে যায় এবং যে কেউ পুরানো commit থেকে দেখতে পারবে!

❌ ভুল পদ্ধতি (এটা করবেন না):

❌ এটা করলে history-তে secret থেকে যাবে!
# ভুল! এই পদ্ধতিতে শুধু latest code থেকে file সরায় # কিন্তু git history-তে সব দেখা যায়! echo ".env" >> .gitignore git add -A git commit -m "remove secret file" git push origin main # ❌ এতে old commits-এ এখনো file আছে

✅ সঠিক পদ্ধতি — এই ক্রমে করুন:

1
🔴 আগে Secrets Rotate করুন (সবচেয়ে জরুরি)
File মোছার আগেই সব exposed credentials পরিবর্তন করুন। পুরানো credentials কাজ করা বন্ধ হলে history-তে থাকলেও কোনো ক্ষতি নেই।
কোথায় Rotate করবেন
# Neon Database: console.neon.tech → Project → Settings → Reset Password # Supabase: app.supabase.com → Settings → Database → Reset Password # Clerk: dashboard.clerk.com → API Keys → Regenerate # OpenAI: platform.openai.com → API Keys → Create new key # Stripe: dashboard.stripe.com → Developers → API keys → Roll key
2
🔧 .gitignore আপডেট করুন
ভবিষ্যতে আর যাতে না যায়।
PowerShell
# .gitignore-এ add করুন echo ".env" >> .gitignore echo ".env.local" >> .gitignore # Verify করুন git check-ignore -v .env
3
🗑️ git filter-repo দিয়ে পুরো history থেকে মুছুন
এই step টাই সবচেয়ে গুরুত্বপূর্ণ। শুধু এটাই পুরো history clean করে।
PowerShell
# ফাইলের নামে space থাকলে quotes দিন git filter-repo --path ".env" --invert-paths --force # বা নির্দিষ্ট ফাইলের জন্য git filter-repo --path "secret-file.md" --invert-paths --force
4
🔗 Remote আবার add করুন
filter-repo automatically remote সরিয়ে দেয়।
PowerShell
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
5
🚀 Force Push করুন
PowerShell
git push origin --force --all git push origin --force --tags
6
✅ Verify করুন
PowerShell — কোনো output না আসলে সফল
# এই command-এ কোনো output না আসলে ফাইল history থেকে মুছে গেছে git log --all --full-history -- "secret-file.md"
7
📧 GitHub Support-এ Cache Purge Request পাঠান
Force push করার পরেও direct commit URL-এ file দেখা গেলে, GitHub Support-এ request পাঠান (নিচের Section ৩ দেখুন)।
💡 মনে রাখুন: শুধু git rm --cached বা git rm করে push করলে শুধু latest code থেকে সরে। কিন্তু পুরানো commits-এ ফাইল থেকে যায়। সবসময় filter-repo ব্যবহার করুন।

🛡️ ৪. আগে থেকে সতর্কতা — .gitignore সেটআপ

প্রতিটা নতুন project শুরুতেই এই কাজগুলো করুন। এটা করলে .env বা sensitive file কখনো GitHub-এ যাবে না।

✅ .gitignore-এ এগুলো অবশ্যই রাখুন:

.gitignore
# Environment files .env .env.local .env.development .env.production .env*.local *.env # Sensitive docs docs/secrets/ *secret* *credentials* *password* # OS files .DS_Store Thumbs.db # IDE .vscode/settings.json .idea/

✅ .gitignore কাজ করছে কিনা চেক করুন:

PowerShell
git check-ignore -v .env

Output দিলে মানে .env ignore হচ্ছে। কোনো output না দিলে .gitignore-এ .env নেই।

🚨 ৫. .env ফাইল GitHub-এ চলে গেলে (বিস্তারিত)

⚠️ এটা সবচেয়ে বিপজ্জনক। দ্রুত নিচের ধাপগুলো অনুসরণ করুন।
1
সাথে সাথে সব secrets rotate করুন — Database password, API keys, JWT secrets সব পরিবর্তন করুন। এটাই সবচেয়ে জরুরি।
2
.env ফাইল .gitignore-এ যোগ করুন (যদি না থাকে):
PowerShell
echo ".env" >> .gitignore echo ".env.local" >> .gitignore git add .gitignore git commit -m "fix: add .env to gitignore"
3
Git tracking থেকে .env সরান (কিন্তু file delete করবেন না):
PowerShell
git rm --cached .env git commit -m "fix: remove .env from tracking"
4
পুরো history থেকে .env মুছুন:
PowerShell
git filter-repo --path .env --invert-paths --force
5
Remote আবার add করুন:
PowerShell
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
6
Force Push করুন:
PowerShell
git push origin --force --all git push origin --force --tags

🗑️ ৬. যেকোনো Unwanted ফাইল GitHub থেকে সরানো

⚡ git filter-repo ইনস্টল না থাকলে আগে করুন: pip install git-filter-repo

একটি নির্দিষ্ট ফাইল সরাতে:

PowerShell — ফাইলের নামে space থাকলে quotes দিন
git filter-repo --path "path/to/secret-file.md" --invert-paths --force

একটি নির্দিষ্ট folder সরাতে:

PowerShell
git filter-repo --path "secret-folder/" --invert-paths --force

একটি নির্দিষ্ট extension-এর সব ফাইল সরাতে:

PowerShell
git filter-repo --path-glob "*.env" --invert-paths --force

কাজ হয়েছে কিনা verify করুন:

PowerShell — কোনো output না আসলে সফল
git log --all --full-history -- "path/to/file.md"

🔍 ৭. Repository-তে Secret আছে কিনা Automated স্ক্যান

Push করার আগে এই tool দিয়ে চেক করুন।

Gitleaks দিয়ে স্ক্যান (recommended):

PowerShell — আগে install করুন: https://github.com/gitleaks/gitleaks
# Winget দিয়ে install winget install gitleaks # পুরো repo স্ক্যান gitleaks detect --source . -v # শুধু staged files স্ক্যান gitleaks protect --staged -v

Git History-তে কোনো keyword আছে কিনা খুঁজুন:

PowerShell
# "password" keyword খোঁজা git log -p --all | Select-String "password" -Context 2 # "DATABASE_URL" keyword খোঁজা git log -p --all | Select-String "DATABASE_URL" -Context 2

GitHub Secret Scanning চালু করুন:

GitHub Repository → Settings → Security → Secret scanning → Enable করুন। GitHub নিজেই আপনাকে notify করবে।
