৭. কোনটা আগে — Business Verification নাকি Access Verification?

উত্তর: প্রথমে Business Verification। Access Verification তার পরে (বা App Review-এর সাথে), প্রায় কখনও আগে নয়।

ক্রম ধাপ কেন এই ক্রম
১ Business Verification Portfolio “Unverified” থাকলে অনেক API, App Review ও billing আটকে যেতে পারে
২ Development-এ পুরো flow test + API calls App Review submit-এর আগে প্রতিটি permission-এ successful call লাগে (৩০ দিনের মধ্যে)
৩ App Review → Advanced access pages_messaging, instagram_manage_messages ইত্যাদি — পাবলিক কাস্টমারের জন্য
৪ Access Verification (Tech Provider) Advanced request করলে Meta form/email পাঠায় — অনেক সময় App Review-এর পরে
৫ App Mode → Live অনুমোদন ও verification শেষে (বা Meta-র নির্দেশ অনুযায়ী)

দুটো verification-এর পার্থক্য (এক লাইনে)

  • Business Verification = “REMOVED নামে কোম্পানি/ব্যবসা আসল কিনা?”
  • Access Verification = “REMOVED কি অন্য ব্যবসার data নিয়ে Tech Provider হিসেবে SaaS চালায়?”

Access Verification আগে শুরু করলেও Business Verification incomplete থাকলে প্রায়ই আটকে যাবে। তাই ডকুমেন্ট প্রস্তুতি শুরু করুন Business Verification দিয়ে।

১০. Access Verification — Tech Provider (বিস্তারিত)

এটা বলতে কী বোঝায়?

REMOVED অন্য ব্যবসার Facebook Page, Instagram, WhatsApp data ব্যবহার করে তাদের পক্ষে AI Chatbot চালায় — এটাই Meta-র ভাষায় Tech Provider।

Business Verification = “REMOVED কোম্পানি real”।
Access Verification = “REMOVED সত্যিই client-দের data দিয়ে SaaS সেবা দেয়, abuse করবে না”।

কখন লাগে?

যখন আপনার App অন্য business-এর data access চায় এবং নির্দিষ্ট permissions-এ Advanced access request করেন — যেমন pages_messaging, instagram_manage_messages। Development mode-এ Standard access দিয়ে শুধু আপনার টিম test করতে পারে; Advanced approve করার পরও API call fail হতে পারে যতক্ষণ Access Verification complete না হয়।

না করলে কী হয়?

Graph API error (সাধারণ message): business Tech Provider হিসেবে verified নয় — অন্য business-এর token দিয়ে call reject।
